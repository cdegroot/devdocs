GET users/:id/on_call

Get a user object with that user's current on-call status. If the on-call object is an empty array, the user is never on-call.

If the start and end of an on-call object are null, then the user is always on-call for an escalation policy level.

Resource URL 

GET https://<subdomain>.pagerduty.com/api/v1/users/:id/on_call

Parameters

Name Type Required Description
include Array No Array of additional details to include. This API accepts contact_methods, and notification_rules.

Examples

Get a single user with current on-call status. 

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    "https://acme.pagerduty.com/api/v1/users/PIJ90N7/on_call"

Note: Content-type of the request must be application/json

Response Status: 200 OK

Get a single user with notification rules and contact methods 

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    --data-urlencode "include[]=contact_methods" \
    --data-urlencode "include[]=notification_rules" \
    "https://acme.pagerduty.com/api/v1/users/:id/on_call"
