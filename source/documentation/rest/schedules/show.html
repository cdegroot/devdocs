GET schedules/:id

Show detailed information about a schedule, including entries for each layer and sub-schedule.

Resource URL

GET https://<subdomain>.pagerduty.com/api/v1/schedules/:id

Parameters

NameTypeRequiredDescription
sinceDateNo

The start of the date range over which you want to return on-call schedule entries and on-call schedule layers.

untilDateNoThe end of the date range over which you want to return schedule entries and on-call schedule layers.
time_zoneTime ZoneNoTime zone in which dates in the result will be rendered. Defaults to account time zone.

Response Fields

NameTypeDescription
schedule_layersArrayAn array of schedule layers.
nameStringThe name of the schedule
final_scheduleObjectThe final schedule layer object.
overrides_subscheduleObjectThe schedule layer object where all the overrides are stored.
time_zoneTime ZoneThe time zone of the schedule.
escalation_policiesArrayAn array of all the escalation policies that uses this schedule.
todayDateThe current day in the schedule's time zone.

The Schedule Layer Object

A schedule is composed of multiple schedule layers.

A layer is composed of a group of people who will rotate through the same shift. In a basic weekly schedule, you create a single layer where each member is on-call for one week, with a set day and time for transferring on-call duty.

When a schedule has multiple layers, the layer can be ordered using the priority field. The layer with the highest priority has precedence over the layers with lower priority. You can use restrictions to control how layers overlap.

The override layer is a special layer where all the override entries are stored.

The final layer is a special layer that contains the result of all the previous layers put together. This layer cannot be edited.

Schedule Layer Fields

NameTypeDescription
rendered_schedule_entriesArrayThis is a list of entries to be rendered for the current time range.
restriction_typeStringPossible values are daily or weekly. This specifies the type of restrictions present on this layer.
restrictionsArrayA list of time restrictions for this layer.
priorityIntegerThe priority of the layer. Layers with higher priority will override layers with a lower priority.
startDateThe effective start date of the layer
endDateThe effective end date of the layer. If missing or null, the layer has no end date.
rendered_coverage_percentageIntegerThe percentage of the time range covered by this layer.
rotation_turn_length_secondsIntegerThe duration of each on-call shift in seconds.
rotation_virtual_startDateThe effective start time of the layer. This can be before the start time of the schedule.
usersArrayAn ordered list of users in this layer. The member_order field controls the order in which the users apear in the rotation.

Example

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    "https://acme.pagerduty.com/api/v1/schedules/PWEVPB6"

Note: Content-type of the request must be application/json
