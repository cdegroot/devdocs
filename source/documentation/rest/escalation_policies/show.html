GET escalation_policies/:id

Get information about an existing escalation policy and its rules.

Resource URL 

GET https://<subdomain>.pagerduty.com/api/v1/escalation_policies/:id

Escalation Policy Response Fields

Name Type Description
id String The ID of the escalation policy.
name String The name of the escalation policy.
num_loops Integer The number of times the escalation policy will repeat after reaching the end of its escalation.
escalation_rules Array A list of the policy's escalation rules in order of escalation. Fields detailed below.
services Array A list of services using this escalation policy. See Services for the fields to expect.

Escalation Rule Response Fields

Name Type Description
id String The ID of the escalation rule.
escalation_delay_in_minutes Integer The amount of time before an incident escalates away from this rule.
rule_object Object
This field is deprecated. See targets instead
An object representing the first target.
targets Array An array of targets which an incident will be assigned to upon reaching this rule. Fields detailed below.

Target Response Fields

Name Type Description
id String The id of the target.
type String A representation of the type of the target. Will be either schedule or user.
name String The name of the target.
email String Only for user targets. The user's email address.
time_zone String Only for user targets. The user's personal time zone.
color String Only for user targets. The color used to represent the user in schedules.

Example

Get an escalation policy 

curl -H "Content-type: application/json" -H "Authorization: Token token=E7px6VVr3PVHZPJq51oa" -X GET -G \
    "https://acme.pagerduty.com/api/v1/escalation_policies/PBZ3BZV"

Note: Content-type of the request must be application/json
